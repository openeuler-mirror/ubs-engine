## 1. 查询拓扑链接信息

### 描述

查询集群内的拓扑连接信息

### 用法

```shell
ubsectl display topo
```
### 输入参数

无

### 约束限制

当前只支持CPU类型节点的连线关系

两个端点之间可能存在多条连接通道

同一条连接通道只显示一遍，按照node信息排序

### 输出信息说明

- 显示内容分为两部分：

1. topo列表：每一行表示两个端点之间一条连接通道
2. topo链路汇总信息：**Total Links（已知总链接数）**；**Available Links（和可用链接数）**

- topo列表中的字段说明：

| 字段名    | 字段描述                                                     | 字段取值 |
| --------- | ------------------------------------------------------------ | -------- |
| node      | 连接起始节点信息。例：computer1(1)<br>当节点脱离集群管理时，显示“-”，表示缺乏信息<br>节点信息由2部分组成：<br>1.括号前部分：主机名<br>2.括号内部分：节点的槽位号 | 字符串   |
| socketId  | 连接起始端的socket信息（OS上标识某一个CPU的socketId值）<br/>当节点脱离集群管理时，显示“-”，表示缺乏信息 | 整数     |
| remote_node | 连接远端节点信息。例：computer1(1)<br>当节点脱离集群管理时，显示“-”，表示缺乏信息<br>节点信息由2部分组成：<br>1.括号前部分：主机名<br>2.括号内部分：节点的槽位号 | 字符串   |
| remote_socketId    | 连接远端的socket信息（OS上标识某一个CPU的socketId值）<br/>当节点脱离集群管理时，显示“-”，表示缺乏信息 | 整数     |

### 示例

- 获取全量各节点信息场景：

```
$ ubsectl display topo
Cputopo：
---------------------------------------------------------------
node          socketId      remote_node      remote_socketId
---------------------------------------------------------------
node-4(4)     36            node-3(3)        1
                            node-1(1)        36
                            node-2(2)        36

node-1(1)     376           node-2(2)        376
                            node-3(3)        376
                            node-4(4)        2

node-4(4)     376           node-1(1)        376
                            node-3(3)        376
                            node-2(2)        2

node-2(2)     36            node-1(1)        36
                            node-3(3)        1
                            node-4(4)        36 
---------------------------------------------------------------
```

## 2. 查询集群内所有节点的信息

### 功能

查询集群内所有节点的信息

### 用法

```shell
ubsectl display cluster
```

### 输入参数

无

### 约束限制

无

### 输出信息说明

| 字段名      | 字段描述                                                     | 取值范围                         |
| ----------- | ------------------------------------------------------------ | -------------------------------- |
| node        | 节点信息。例：computer1(1)<br/>节点信息由2部分组成：<br/>1.括号前部分：主机名<br/>2.括号内部分：节点的槽位号 | 字符串                           |
| role        | 节点在集群中的角色                                           | 可选值：[master\|standby\|agent] |
| bonding-eid | 节点bonding配置的eid                                         | 字符串                           |

### 示例

#### 场景1：集群各节点健康状态

```         
$ ubsectl display cluster
-----------------------------------------------------------------------------

-----------------------------------------------------------------------------

computer02(2)         standby       4245:4944:0000:0000:0000:0000:0200:0000 
computer03(3)         agent         4245:4944:0000:0000:0000:0000:0300:0000 
-----------------------------------------------------------------------------

```
