# 构建指导

## 1 拉取源码

```shell
git clone https://atomgit.com/openeuler/ubs-engine.git
```

## 2 构建

### 2.1 构建依赖

ubs-engine构建依赖信息记录在spec文件（[ubs-engine.spec](../../ubs-engine.spec)）中。
由于当前依赖的ubs-comm-devel版本只发布在openEuler 24.03 LTS SP3/SP4，通过`cat /etc/openEuler-release`确认编译系统版本：

- 如果编译系统版本是`openEuler release 24.03 (LTS-SP3)/(LTS-SP4)`，参考[2.1.1 openEuler release 24.03 (LTS-SP3)/(LTS-SP4) 依赖安装说明](#211-openeuler-release-2403-lts-sp3lts-sp4-依赖安装说明)
- 如果编译系统版本不是`openEuler release 24.03 (LTS-SP3)/(LTS-SP4)`，参考[2.2.2 其他openEuler版本依赖安装说明](#222-其他openeuler版本依赖安装说明)

#### 2.1.1 openEuler release 24.03 (LTS-SP3)/(LTS-SP4) 依赖安装说明

- **方式一：DNF一键安装**

    ```shell
    # 安装spec中指定的BuildRequires
    cd ubs_engine
    dnf builddep -y ubs-engine.spec
    ```

- **方式二：yum手动安装**

    ```shell
    yum install -y libboundscheck libxml2-devel numactl-libs openssl-devel rapidjson-devel patch libvirt-devel kernel-devel 'cpp-httplib-devel >= 0.40.0' 'ubs-comm-devel >= 1.0.1-7' ...
    ```

#### 2.2.2 其他openEuler版本依赖安装说明

> 注意：使用其他openEuler版本时,安装到的三方库依赖版本可能过低,导致编译失败。

执行以下命令安装依赖：

```shell
yum install -y libboundscheck libxml2-devel numactl-libs openssl-devel rapidjson-devel patch libvirt-devel cpp-httplib-devel kernel-devel
```

源码编译ubs-comm-devel和ubs-comm-lib：

```shell
# 安装基础工具链和 rpm 构建工具
sudo dnf install -y git rpm-build rpmdevtools gcc gcc-c++ make
mkdir -p /tmp/src
# 下载 ubs-comm 源码
git clone -b openEuler-24.03-LTS-SP4 https://gitcode.com/src-openeuler/ubs-comm.git /tmp/src/ubs-comm
# 初始化 rpmbuild 目录结构
rpmdev-setuptree
# 复制 ubs-comm 源码中的 spec 文件到 rpmbuild 目录
cp /tmp/src/ubs-comm/ubs-comm.spec ~/rpmbuild/SPECS/
cp -rf /tmp/src/ubs-comm/ubs-comm-*.tar.gz ~/rpmbuild/SOURCES/
# 安装 ubs-comm.spec 中指定的 BuildRequires
sudo dnf builddep -y ~/rpmbuild/SPECS/ubs-comm.spec
# 构建 ubs-comm 包
rpmbuild -ba ~/rpmbuild/SPECS/ubs-comm.spec
# 安装构建得到的 rpm 包
# 架构为aarch64
sudo dnf install -y ~/rpmbuild/RPMS/aarch64/ubs-comm-lib-*.rpm
sudo dnf install -y ~/rpmbuild/RPMS/aarch64/ubs-comm-devel-*.rpm
# 架构为x86_64
sudo dnf install -y ~/rpmbuild/RPMS/x86_64/ubs-comm-lib-*.rpm
sudo dnf install -y ~/rpmbuild/RPMS/x86_64/ubs-comm-devel-*.rpm
```

### 2.3 执行构建

```shell
# 执行 Release 构建（没有调试信息，-O2 优化）
bash build.sh

# 执行 Debug 构建（附加调试信息）
bash build.sh -D

# 执行 RelWithDebInfo 构建（附加调试信息，-O2 优化）
bash build.sh -T RelWithDebInfo

# 执行 MinSizeRel 构建（没有调试信息，二进制文件最小构建，-Os 优化）
bash build.sh -T MinSizeRel
```

## 3 形成rpm包

```shell
# 构建项目，并打包成 rpm 文件输出到项目顶层目录的 output/ 下。
bash build.sh package
```

产物如下：

```path
└── output                                                            # 打包输出文件目录
    ├── ubs-engine-<version>-<release>.aarch64.rpm                    # 主程序包
    ├── ubs-engine-client-libs-<version>-<release>.aarch64.rpm        # 客户端运行时库
    ├── ubs-engine-client-devel-<version>-<release>.aarch64.rpm       # 客户端开发包
    ├── python3-ubs-engine-<version>-<release>.aarch64.rpm            # Python API 包
    ├── ubs-engine-rmrs-<version>-<release>.aarch64.rpm               # 内存池化插件
    ├── ubs-engine-ucache-<version>-<release>.aarch64.rpm             # ucache 插件
    ├── ubs-engine-virtagent-<version>-<release>.aarch64.rpm          # 虚拟化代理插件
    ├── ubs-engine-processmem-<version>-<release>.aarch64.rpm         # processmem 插件
    └── ubs-engine-debuginfo-<version>-<release>.aarch64.rpm          # 调试信息
```

子包依赖关系详见 [部署说明中的依赖分组说明](./部署说明.md#11-依赖分组说明)。
