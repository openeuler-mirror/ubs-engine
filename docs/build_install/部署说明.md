# 部署说明

## 1 概述

ubs engine的发布件包含：开发包、运行包

ubs engine的部署分为开发包部署和运行包部署

- 开发包：主要用于支持开发者使用ubs engine提供的接口，进行业务开发

- 运行包：主要用于在生产环境中与周边ubfm等模块交互，提供ub池化资源管理能力

ubs engine的发布件，由以下四个 RPM 包组成：

| RPM包                                                       | 说明                                |
| ----------------------------------------------------------- | ---------------------------------- |
| ubs-engine-\<version\>-\<release\>.aarch64.rpm              | 主程序包，包含服务、CLI、配置等       |
| ubs-engine-client-libs-\<version\>-\<release\>.aarch64.rpm  | 客户端运行时库（供第三方程序动态链接） |
| ubs-engine-client-devel-\<version\>-\<release\>.aarch64.rpm | 开发包（含头文件与静态库）            |
| python3-ubs-engine-\<version\>-\<release\>.aarch64.rpm      | python API 接口 |

## 2 安装开发包

### 2.1 环境要求

业务开发环境中部署即可，无特殊要求

### 2.2 安装命令

- 方式一

  ```bash
  # 安装开发包（用于集成开发）
  # 注：需要系统配置了openEuler release 24.03 (LTS-SP3)镜像源
  sudo dnf install -y ubs-engine-client-devel
  # 安装python 模块（可选，使用UBSE Python API时需要安装）
  sudo dnf install -y python3-ubs-engine
  ```

- 方式二

  ```bash
  # 安装开发包（用于集成开发）
  # 通过rpm包安装开发包
  sudo dnf install -y ubs-engine-client-devel-<version>-<release>.aarch64.rpm
  # 安装python 模块（可选，使用UBSE Python API时需要安装）
  sudo dnf install -y python3-ubs-engine-<version>-<release>.aarch64.rpm
  ```

### 2.3 安装结果

**ubs-engine 开发包安装结果：**

| 文件/目录                      | 其它说明                                     |
| ------------------------------ | -------------------------------------------- |
| `/usr/include/ubse`            | 目录下头文件（`*.h`）权限：`644`             |
| `/usr/lib64/libubse-client.so` | 软链接，指向`/usr/lib64/libubse-client.so.1` |
| `/usr/lib64/libubse-client.a`  | 二进制静态库                                 |

**ubs-engine Python API 包安装结果：**

| 文件/目录                      | 其它说明                                     |
| ------------------------------ | -------------------------------------------- |

**安装注意事项**

开发包需要依赖客户端包，即ubs-engine-client-devel依赖ubs-engine-client-libs，安装ubs-engine-client-devel前需要安装ubs-engine-client-libs。

## 3 安装运行包

### 3.1 系统要求

* <b>操作系统：</b> openEuler 24.03 LTS 或更高版本
* <b>CPU架构：</b> aarch64
* <b>内存：</b> ≥ 64GB
* <b>磁盘：</b> SSD，IOPS 500MB/s
* <b>芯片互联：</b> UB
* <b>网卡：</b>可选依赖（可选使用TCP辅助UB建链，默认采用UB自举建链）
* <b>用户权限：</b> 安装与管理需 <code>root</code> 权限

### 3.2 节点规划

* 所有集群节点均需安装 <code>ubs-engine</code> 主包。

* 建议提前规划节点 IP 地址列表（用于 TCP 模式）

### 3.3 依赖运行时库

ubs-engine运行依赖信息记录在spec文件（[ubs-engine.spec](../../ubs-engine.spec)）中。
运行依赖所需系统库，通常由包管理器自动安装。

- **安装注意事项**
  - 非高安场景下，UBSE与UBM使用UDS通信，需要将ubse用户加到ubm_nuds用户组中，该用户组由UBM服务创建，如果UBM服务未在UBSE前安装，ubse用户可能无法加入ubm_nuds用户组，导致UBSE服务与UBM服务通信异常。
  - UBSE需要调用ubturbo接口，ubturbo接口有权限校验，需要将ubse用户加到ubturbo用户组中，该用户组由ubturbo服务创建，如果ubturbo服务未安装，ubse用户可能无法加入ubturbo用户组，导致UBSE服务调用ubturbo接口异常。

### 3.4 RPM 包安装步骤

- 方式一

  ```bash
  # 注：需要系统配置了openEuler release 24.03 (LTS-SP3)镜像源
  # 安装主程序包
  sudo dnf install -y ubs-engine
  # 安装客户端运行时库（第三方集成必需）
  sudo dnf install -y ubs-engine-client-libs
  # 安装python 模块（可选，使用UBSE Python API时需要安装）
  sudo dnf install -y python3-ubs-engine
  ```

- 方式二

  ```bash
  # 通过rpm包安装运行包
  # 安装主程序包
  sudo dnf install -y ubs-engine-<version>-<release>.aarch64.rpm
  # 安装客户端运行时库（第三方集成必需）
  sudo dnf install -y ubs-engine-client-libs-<version>-<release>.aarch64.rpm
  # 安装python 模块（可选，使用UBSE Python API时需要安装）
  sudo dnf install -y python3-ubs-engine-<version>-<release>.aarch64.rpm
  ```

  > **用户组配置：**
  > 
  > 安装时默认随机分配 gid/uid。容器场景需根据用户命名空间配置采取不同策略：
  > - **不隔离用户命名空间**（如 Kubernetes 1.27 以下）：将容器进程在宿主机上的实际用户加入 UBSE 用户组，通过组权限通信。
  > - **启用用户命名空间隔离**（如 Docker/Podman userns-remap）：在宿主机预制映射目标用户并加入 UBSE 用户组，配置容器引擎映射至该用户。
  > 
  > 指定固定 gid/uid 方法：安装前设置环境变量（推荐 ≤ 999，避免冲突）：
  > ```bash
  > export UBSE_USER_UID=983
  > export UBSE_USER_GID=983
  > ```

## 4 通信配置

### 4.1 通信模式

UBSE 默认采用 URMA+TLS 证书的安全通信方式，支持以下四种通信模式：

| 通信模式              | 安全性 | 性能                   | 适用场景                   |
| --------------------- | ------ | ---------------------- | -------------------------- |
| URMA+TLS（默认推荐）  | 高     | 高性能、低延迟、零拷贝 | 生产环境、高性能计算       |
| TCP+TLS               | 高     | 标准 TCP，性能适中     | 生产环境、无 URMA 硬件支持 |
| URMA（不带 TLS）      | 低     | 高性能、低延迟、零拷贝 | 可信环境、开发测试         |
| TCP（不带 TLS）       | 低     | 标准 TCP，性能适中     | 可信环境、开发测试         |

> **安全提示：** 不开启 TLS 的 URMA 通信和纯 TCP 通信，没有认证和加密，存在仿冒、消息泄漏等安全风险，请确保 UBSE 运行的软硬件环境是可信的。

**证书导入方法：** 使用 TLS 证书通信时，需先导入证书，具体操作请参考 [证书管理](../cli/ubsectl_cert.md)。

**证书配置说明：** 默认开启TLS证书认证，若需要关闭则修改配置文件相关参数关闭 TLS 证书认证：

1. 编辑配置文件：

    ```bash
    sudo vi /etc/ubse/ubse.conf
    ```

2. 在 `[ubse.rpc]` 配置段中，设置 `cert.use` 参数：

    ```ini
    [ubse.rpc]
    cert.use=false
    ```

    - `cert.use=true`：开启 TLS 证书认证（默认值）
    - `cert.use=false`：关闭 TLS 证书认证

3. 重启服务使配置生效：

    ```bash
    sudo systemctl restart ubse
    ```

### 4.2 URMA 通信模式（推荐）

URMA 通信模式具有以下特点：

* 安装后直接启动服务，无需任何配置。
* 节点通过 URMA 自动发现并建立连接。
* 适用于具备 URMA 硬件支持的环境。

#### 4.2.1 前置依赖检查

在安装 UBSE 前，请先检查 URMA 和 UBS-COMM 用户态组件是否已安装：

```bash
# 检查 URMA 用户态是否安装
rpm -qa | grep umdk

# 检查 UBS-COMM 用户态是否安装
rpm -qa | grep ubs-comm
```

若上述命令无输出或输出为空，说明相关组件未安装，需参考 URMA 和 UBS-COMM 组件的安装方法进行安装。

#### 4.2.2 启动 ubs engine 服务

```bash
sudo systemctl start ubse
sudo systemctl enable ubse
```

### 4.3 TCP 通信模式

#### 4.3.1 修改配置

1. 编辑配置文件：

    ```bash
    sudo vi /etc/ubse/ubse.conf
    ```

2. 修改以下配置项(默认无此配置项,打开此配置时，使用tcp通信，否则默认使用urma通信)，示例如下：

    ```ini
    [ubse.rpc]
    cluster.ipList=192.168.100.100-192.168.100.102
    ```

> 192.168.100.100-192.168.100.102等同于192.168.100.100,192.168.100.101,192.168.100.102

#### 4.3.2 重启 ubs engine 服务

```bash
sudo systemctl restart ubse
```

## 7 升级与卸载

**RPM 配置文件升级规则：**

- 配置文件在新的rpm包里相对之前的rpm有变化，且在本地没有被修改过。升级时新rpm包里的配置文件会替换覆盖旧的文件。
- 配置文件在新的rpm包里相对之前的rpm有变化，且在本地被修改过。升级时旧文件保持不变，新rpm包里的配置文件并重命名为xx.rpmnew，例如"/etc/ubse/ubse.conf.rpmnew"。
- 配置文件在新的rpm包里相对之前的rpm没有变化，且在本地没有被修改过。升级时新rpm包里的配置文件会替换旧的文件（旧文件被删除）。
- 配置文件在新的rpm包里相对之前的 rpm 没有变化，且在本地被修改过。升级时新rpm包里的配置文件不会覆盖旧的文件，旧文件保持不变。
- 软件卸载，被修改过的配置文件会以.rpmsave后缀备份。备份的配置文件不删除，如以.rpmnew或者.rpmsave后缀的备份的配置文件。


### 7.1 升级

```bash
# 通过rpm包进行升级
sudo dnf update -y *ubs-engine*.rpm
# 配置openEuler release 24.03 (LTS-SP3)镜像源进行升级
sudo dnf update -y ubs-engine ubs-engine-client-libs ubs-engine-client-devel python3-ubs-engine
```

### 7.2 卸载

```bash
sudo dnf remove -y ubs-engine ubs-engine-client-libs ubs-engine-client-devel python3-ubs-engine
```

> 卸载不会删除 <code>/var/lib/ubse/data</code> 中的数据，如需清理请手动删除。

## 8. 常见问题

### Q1: 服务启动失败，提示权限错误？

* 检查 <code>/var/lib/ubse/data</code> 和 <code>/var/lib/ubse</code> 所属用户是否为 <code>ubse:ubse</code>。

* 使用 <code>chown -R ubse:ubse /var/lib/ubse /var/log/ubse</code> 修复。其中权限设置如下：

  | 路径               | 权限        |
  | ------------------ | ----------- |
  | /var/lib/ubse      | ubse:ubse 750 |
  | /var/lib/ubse/data | ubse:ubse 750 |

### Q2: CLI 命令补全无效？

* 安装 <code>bash-completion</code>：<code>sudo dnf install bash-completion</code>

* 重新登录或执行：<code>source /etc/bash_completion.d/cli_commands.sh</code>

### Q3: 如何查看当前通信模式？

* 使用命令快速查看配置：

```bash
grep -E "cluster.ipList" /etc/ubse/ubse.conf
```

输出示例:

```text
# cluster.ipList=192.168.100.100-192.168.100.102,192.168.100.104
```

* 若显示 cluster.ipList 行首有 # 注释符 , 当前使用 URMA 模式
* 若显示 cluster.ipList 行首无 # 注释符, 当前使用 TCP 模式
