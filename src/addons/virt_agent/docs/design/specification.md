# UBS VirtAgent规格约束

## 内存超分逃生

- 内存超分最大借用比例80%. 内存超分比例小于等于25%, 虚拟机性能劣化不超过5%; 内存超分比例超过25%, 性能劣化不做保证.
- 目前内存超分, 针对2M大页场景
- 内存借用速率在开启内存预留与内存预上线特性后, 在预留内存范围内的内存借用, 内存借用速率满足不低于50ms/GB; 详细参考: [内存池化借用性能规格设计.md](../../../../../docs/design/%E5%86%85%E5%AD%98%E6%B1%A0%E5%8C%96%E5%80%9F%E7%94%A8%E6%80%A7%E8%83%BD%E8%A7%84%E6%A0%BC%E8%AE%BE%E8%AE%A1.md)
- 内存借用/归还特性, 依赖UBS RMRS与UBS Engine提供的内存借用/归还能力. 需要满足相关组件对借用功能的约束; 当前实现, 不支持跨平面借用[^跨平面]
  - UBS Engine接口约束描述, 请参考[libubse_mem.md](../../../../../docs/api/libubse_mem.md)

## 内存碎片整理

- 内存碎片场景, 最大内存借用比例不超过25%, 虚拟机性能劣化不超过5%
- 目前内存碎片, 针对2M大页场景
- 内存借用/归还特性, 依赖UBS RMRS与UBS Engine提供的内存借用/归还能力. 需要满足相关组件对借用功能的约束; 当前实现, 不支持跨平面借用[^跨平面]
  - UBS RMRS接口约束描述, 请参考[Design_docs_Reference.md](../../../rmrs/docs/Design_docs_Reference.md)

## 虚拟机确定性热迁移

- 确定性热迁移虚机, 内存规格最小不低于4G
- 确定性热迁移, 最大超时时间, 不超过60秒
- 确定性热迁移, 本质是通过内存借用, 借用UB通道, 实现虚机内存搬迁. 依赖UBS Engine提供的内存借用/归还接口.

## 容器化内存借用

### 特性约束

- 用户通过节点标签配置节点内存超分比例, pod可以通过labels配置容器进程本地远端的比例. 如果按照25%配置, 容器进程的本地内存/远端内存配比为3:1
- 该方案适用于单个pod多个容器多进程, 内存告警可以出发多次借用; 一个进程用远端内存, 目标态**单进程只使用一个远端NUMA**(跟随可靠性约束), 中间态可能会出现使用多个NUMA内存(远端NUMA故障场景)
- 该方案适用于容器绑单NUMA场景和容器不绑定NUMA场景, 支持4K标准页与64K页场景, **因此需要配置不支持应用直接申请远端内存**(需要提前配置), 配置方法如下:
  - 修改`/etc/default/grub`文件新增`numa_remote=nofallback`启动参数
- 该方案支持运行时**containerd**, 其他运行时软件暂不支持
- 该方案不涉及节点借出内存来源, 用户节点规划时需要考虑, 节点内存告警后, 借不到内存, 容器进程可能OOM, 本方案提供内存超分和逃生机制, 并不保证完全避免OOM
- 推荐应用绑NUMA场景, 借用内存打标签的容器都可以使用, 如果有严格限制不能跨NUMA的进程不支持借用远端内存
- **容器无感借用保证性能下降5%以内**
- **没有配置资源`request/limit`的进程不在管控范围内, 主机进程不在管控范围内**
- 该方案针对裸机容器场景, 不使用虚机容器场景
- k8s有自身的保护机制, 该方案不支持OOM紧急借用
- 容器绑定单NUMA场景支持NUMA级告警, 容器不绑定NUMA场景支持节点级水线告警
- 当前硬件形态和约束, 依赖其他节点提供连续空间可借出内存

### Kubernetes对绑核和内存超分的限制约束

- 该方案需要kubernetes v1.31.2及以上版本(k8s版本需要支持kubelet MemoryManager特性)
- 同一个节点支持不绑定NUMA场景和混合部署场景(绑定NUMA+不绑定NUMA); 混合部署场景, 只支持绑定NUMA的POD内存借用;
- 节点采集数据存收kubernetes的数据采集计算规则, 如果和OS上采集数据有差异, 以kubernetes采集的数据为准, 遵守kubernetes的生态规范, 如果kubelet没有相关数据, 则自行采集
- 容器绑定NUMA场景, 不支持内存超分.

### 资源规格

kube-matrix-agent: CPU资源: 0.5U, 内存资源: 200M(limit), 管理150个pod(k8s默认支持110个, 支持最多150需要修改配置, k8s的能力不属于该需求交付范围), 最多300个容器, 最多300个进程

### 性能标准测试场景

数据库容器绑定NUMA场景: 

- 基线组: 不使用远端内存容器规格8U40G(40G本地内存)
- 测试组: 使用25%远端内存, 容器规格8U40G(30G本地内存, 10G远端内存)

单个容器性能下降<=5%

[^跨平面]: 物理硬件拓扑中, 物理芯片之间, 无直连的UB通道. 当前发布的硬件形态中, 单节点包含4 NUMA(0~3); 其中相邻节点的 0~1 NUMA存在UB通道, 为同平面; 2~3 NUMA存在UB通道, 为同平面
